# 功能岛 2：AI 例句建议

## 1. 前因

词典例句在很多场景下不够用：

- 数量少
- 语境单一
- 不一定贴近学习目标

而对背词用户来说，例句不是装饰信息，而是帮助理解词义、固定搭配、语境回忆的核心材料。

当前仓库已经具备基础例句生成能力，因此这个功能岛可以在现有能力上升级为可采纳的学习材料模块。

## 2. 功能目标

为单词详情页提供可采纳的 AI 例句建议，帮助用户在词典原始例句之外获得更多可学习语境。

生成的例句应强调：

- 自然
- 可学习
- 与当前词义相关
- 尽量覆盖不同使用场景

## 3. 范围

覆盖内容：

- 生成多条例句建议
- 例句建议列表展示
- 单条例句采纳与编辑
- 与词典例句的视觉区分

不覆盖内容：

- 批量例句生成
- 例句自动翻译
- 复杂风格选择器

## 4. 总览实施计划

### 4.1 建议内容

默认生成 3 条例句建议，尽量覆盖：

- 简单直接的基础例句
- 更贴近常见使用场景的例句
- 能体现搭配或常见句型的例句

### 4.2 展示方式

在详情页单独呈现：

- 词典例句
- AI 例句建议

避免用户误以为 AI 内容来自词典原文。

### 4.3 用户动作

用户可对每条例句执行：

- 采纳
- 编辑后采纳
- 丢弃
- 重新生成

### 4.4 导出作用

被采纳的 AI 例句将作为卡片背面的附加例句内容，与词典例句共同构成学习材料。

## 5. 验收点

- 用户可以为单个词条生成多条例句建议
- 例句建议与词典例句有明确区分
- 用户可逐条采纳、编辑、拒绝例句
- 被采纳例句可持久化
- 被采纳例句可出现在卡片预览和导出结果中
- 重生成不会误覆盖用户已采纳的例句

## 6. 并行开发边界

该岛依赖：

- AI 建议层

除此之外，与 Recall Card Draft、易错点、搭配用法都相对独立，可单独并行开发。
